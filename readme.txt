== Installation ==

1. Upload the plugin files to `/wp-content/plugins/lfcc-leave-management/`
2. Activate the plugin through the 'Plugins' screen in WordPress
3. Go to Leave Management ‚Üí Settings to configure the plugin
4. Set up your subdomain (optional but recommended)
5. Configure email settings for notifications
6. Create user accounts or enable registration
7. Customize email templates as needed

= Subdomain Setup =

1. Create a CNAME DNS record pointing your subdomain to your main domain
2. Configure the subdomain URL in plugin settings
3. Ensure your SSL certificate covers the subdomain
4. Test access at your configured subdomain

= Email Configuration =

1. Go to Leave Management ‚Üí Settings ‚Üí Email
2. Configure your SMTP server details
3. Test email functionality
4. Customize email templates with the WYSIWYG editor

== Frequently Asked Questions ==

= Do I need a subdomain to use this plugin? =

No, the subdomain functionality is optional. The plugin works perfectly without it, but a subdomain provides a more professional experience for your staff.

= Can users edit their approved leave requests? =

Yes, if enabled in settings. When users edit approved requests, they automatically reset to pending status for re-approval.

= Does the plugin count weekends as leave days? =

This is configurable in the settings. You can choose whether weekends count toward leave calculations.

= Can I customize the email templates? =

Absolutely! The plugin includes a WYSIWYG editor with 15+ variables for complete email customization.

= Is the plugin mobile-friendly? =

Yes, both the admin interface and frontend are fully responsive and mobile-optimized.

= Can I export leave data? =

Yes, you can export users, leave requests, and balance data to Excel/CSV format.

= What user roles are supported? =

The plugin supports Staff, HR, and Admin roles with different permission levels.

= Can I set different leave allocations per user? =

Yes, each user can have individual leave allocations for all leave types.

== Privacy Policy ==

This plugin stores user data including names, email addresses, leave requests, and related information in your WordPress database. All data remains on your server and is not transmitted to external services except for configured email notifications.
